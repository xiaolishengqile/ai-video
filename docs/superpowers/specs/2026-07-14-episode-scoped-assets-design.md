# 分集资产目录与剧本分镜精准绑定设计

日期：2026-07-14

## 1. 目标

用户先创建项目，再按角色、场景、道具三类上传图片目录。目录中的“第 N 集角色图 / 场景图 / 道具图”决定资产只属于该集。AI 解析剧本和生成分镜时只能使用当前集已上传的资产，从而避免跨集误用和凭空重复创建资产。

用户选择的规则是：即使同一主角在多集出现，也会在每一集目录中单独上传；它们必须是独立资产，不共享、不自动复用。

## 2. 导入目录规则

用户仍在导入界面显式选择资产类型 `character`、`scene` 或 `prop`；文件夹名只负责解析剧集范围。

导入文件的相对路径必须包含且仅包含一个剧集目录段：`第 N 集`、`第N集`、`第八集`、`第十五集` 均合法。系统扫描路径段并将中文或阿拉伯数字转换为正整数 `episodeNumber`。

示例：

```text
道具图/第八集道具图/能量核心.png
道具图/第八集道具图/修复装置.png
```

导入为：

```text
能量核心   type=prop, episodeNumber=8
修复装置   type=prop, episodeNumber=8
```

文件名有语义，因此首期以归一化文件名作为资产名称；不接入图片视觉识别或 OCR。现有主资产/变体识别规则仍适用，但父资产查找必须限定在同一项目、同一剧集、同一类型内。

缺少集数目录、路径中出现两个不同集数、或集数不合法的文件在导入预览和导入结果中标记为失败，不创建范围不明确的资产。

## 3. 数据模型与兼容性

在 `afv_asset` 新增可空整数字段 `episode_number`。新分集导入创建的资产必须写入该字段；同名资产的唯一键改为：

```text
project_id + episode_number + type + normalized_name + deleted
```

因此“凌炽”可以同时存在于第 1 集和第 2 集，但第 1 集不能再导入同类型、同名的第二个“凌炽”。每个 `AssetItem` 通过 `asset_id` 继承其剧集归属，无需增加映射表。

历史资产保留 `episode_number = NULL`，不回填、不删除。严格分集 AI 流程不会将这些未归集资产作为候选；用户可按新目录重新导入。

## 4. 资产查询边界

新增“查询本集资产目录”能力，输入 `projectId` 和 `episodeNumber`，只返回该集角色、场景、道具及其可用子资产：ID、名称、类型、封面、描述、标签和初始/变体子资产 ID。

全项目目录查询保留给资产管理界面、历史功能和快照用途；剧本解析与分镜 Agent 不再使用它来匹配分集资产。

服务层提供按 `projectId + episodeNumber + type + normalizedName` 查询资产的确定性接口。所有分集解析和镜头保存都通过该接口验证资产范围。

## 5. 剧本解析结果

解析第 N 集时，Agent 首先查询第 N 集资产目录，再分析该集的每一个场次。它不会把目录中全部资产绑定到每个场次，而是只绑定文本中实际出场的资产。

每个场次保存的 `entityManifest` 包含：

- 实际出场角色的资产与初始子资产 ID；
- 该场使用的场景资产与初始子资产 ID；
- 实际出现、影响构图或连续性的道具资产与初始子资产 ID；
- 资产来源 `uploaded_episode_catalog`；
- 未能在本集目录精确匹配的实体名称与原因。

未匹配实体仍保留在场次文本描述中，但不会创建 AI 资产、不会借用其他集同名资产、不会关联错误 ID。这样用户可补传资产后重新解析该集。

## 6. 分镜解析与镜头关联

生成第 N 集分镜时，Agent 查询本集剧本场次；场次携带已解析的 `entityManifest`。镜头默认继承场次的核心角色、场景、道具，再依据实际入画范围决定最终关联：

- 全景、场景建立镜头：关联该场核心场景及实际入画角色/道具。
- 道具特写：关联对应道具；可排除不入画的角色和场景。
- 人物特写：关联角色；场景或道具不入画时明确排除。
- 没有出现的资产不关联；不会将本集目录的全部资产塞入每个镜头。

`save_storyboard_scene_shots` 在保存前校验每个显式或继承资产都属于当前 `scriptEpisode` 的 `episodeNumber`；跨集 ID 直接返回参数错误。最终镜头持久化的仍是 AssetItem ID，因此首尾帧、图像生成和 UI 展示可直接使用正确图片。

## 7. 前端体验

导入预览显示每个文件解析出的“第 N 集”、资产类型、资产名、主资产/变体归类和失败原因。资产列表、场次详情和镜头详情显示“第 N 集”标签，帮助用户检查资产范围和实际绑定来源。

首期不提供项目通用资产、手动跨集关联、图片视觉命名或自动迁移历史未归集资产。

## 8. 错误处理

- 目录无集数、集数冲突、格式不合法：该文件失败，其他文件继续导入。
- 同集同类型同名资产：主资产跳过；同集同父资产同名变体跳过。
- 场次文本提到但本集目录没有的实体：记录未匹配，不跨集回退，不自动创建。
- 分镜提交跨集资产 ID：整个镜头保存请求失败并说明资产和期望集数。
- 访问无权项目、剧集与项目不匹配：返回权限或参数错误。

## 9. 测试与验收

单元测试覆盖：

1. `第八集`、`第 8 集`、`第十五集` 的路径解析，以及缺失/冲突集数失败。
2. 同名资产在不同集可创建、同集同类型同名被拒绝、变体不会跨集归并。
3. 本集目录查询不返回其他集或未归集资产。
4. 场次 manifest 只绑定当前集精确匹配的角色、场景、道具；缺失实体不创建资产。
5. 镜头默认继承实际场次资产，跨集显式资产与跨集继承均被拒绝。

验收场景：用户导入“第八集道具图/能量核心.png”，解析第八集后含“能量核心”的场次绑定该道具；第九集即使有同名或其他道具，也不能关联第八集的 AssetItem。
