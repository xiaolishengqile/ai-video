# 分集本地资产解析与分镜参考闭环设计

## 目标

让完整剧本解析和分镜生成只使用当前剧集的用户资产；AI 从当前集的小候选集合中选择角色、场景、道具及其子图，缺失时才创建当前集无图占位资产。最终镜头绑定的子图必须能被首尾帧和视频生成流程作为参考图读取。

不使用图片 embedding，不引入新依赖，也不跨集搜索资产。

## 现状与根因

完整剧本解析先读取全项目资产，再用 `batch_create_assets` 为整部剧创建主资产；该工具创建的资产没有 `episodeNumber`。随后场次和快照又只查询当前集资产，导致项目级新资产和带后缀的已上传图片均不能命中，最终产生重复无图占位资产。

现有快照创建发生在场次解析前，但场次解析仍可能补建资产，因此快照不是最终目录。分镜预处理可创建变体记录，却不会生成图片；无图变体不能作为视觉参考。

## 范围与不变量

- 每次候选搜索仅查询 `projectId + episodeNumber + assetType`，不回退到其他集或项目级资产。
- 角色/场景/道具主资产的身份匹配与镜头子图选择分离。
- 匹配仅依据文本名称：精确名称、已知结尾词归一化、明确别名；不做任意子串匹配，不读取图片像素。
- 多个同分候选不得静默选择；返回歧义，AI 必须显式选择候选或创建当前集占位资产。
- 自动补建资产和任何新变体必须带当前 `episodeNumber`；无 `imageUrl` 的项目只能表示实体，不能宣称是视觉参考图。
- 分镜只消费在所有场次绑定和变体准备完成后创建的不可变分集快照。
- 每个镜头保留一个主场景 `sceneAssetItemId` 以兼容既有流程；可通过有序 `sceneAssetItemIds` 增加场景参考，首项始终是主场景。

## 名称匹配策略

输入名称与候选主资产名称统一进行空白归一化。其后仅剥离位于**末尾**的已知视觉后缀：`表情图`、`三视图`、`正面`、`侧面`、`背面`、`细节`、`多机位`、`六面展开图`。服装、战斗形态、昼夜和内外景属于可能改变身份或外观的业务语义，不能自动剥离。

匹配按以下顺序评分：

1. 原始归一化名称相等：`exact`。
2. 去已知后缀后的名称相等：`suffix_normalized`。
3. 候选资产的显式别名相等：本次不新增持久化别名字段，预留为未来扩展。

候选搜索返回全部最高分候选（最多五个）和其子资产。唯一最高分候选可自动复用；多个最高分候选返回 `ambiguous`，不会创建或绑定错误资产。

## 目标流程

```text
用户按“第 N 集”导入图片
  → 完整剧本 Agent 仅保存剧本元信息和分集原文
  → 每集场次 Agent：解析实体 → 搜索本集候选 → 选择/校验主资产 → 保存 entityManifest
  → 本集变体预处理：只查看 entityManifest 已引用主资产；保留已有有图子项，必要时创建“待补图”变体
  → 创建本集最终资产快照
  → 分集分镜 Agent：从快照选择每镜头的 1..N 个子资产
  → 首尾帧/视频 Agent：读取镜头已绑定且有图的子资产，按模型能力传参考图
```

### 完整剧本解析

完整剧本 Agent 不再调用 `list_project_assets` 或 `batch_create_assets`。它保存标题、梗概、分集原文；`charactersJson` 可先保存无资产 ID 的文本人物表。主资产绑定由分集场次 Agent 负责，避免在剧集号尚未确定时创建无范围资产。

### 分集候选搜索与场次绑定

新增 `search_episode_asset_candidates` 工具。输入为 `projectId`、`scriptEpisodeId`、`assetType` 与 `name`，输出当前集候选主资产、匹配方式、子资产 ID/名称/图片 URL。工具只做候选与排序，不修改数据。

`resolve_scene_entity_manifest` 允许实体携带可选 `selectedAssetId`。若提供，后端必须验证该资产属于当前项目、当前集且类型一致；若未提供，服务仅在唯一候选时自动复用。无候选的 core/supporting 实体才补建当前集无图资产。`source` 精确记录 `matched_exact`、`matched_suffix_normalized`、`matched_selected`、`ambiguous_episode_catalog` 或 `auto_created_episode_catalog`。

分集场次提示词先搜索每个独特实体，再将候选中的 `assetId` 回填给 resolve。AI 只比较当前集候选的图片名称；不把全项目文件名加入上下文。

### 快照与分镜

场次 Agent 不再在开头依赖快照。所有分集场次保存后，完整解析主 Agent 为每集创建一次快照；该快照才是分镜的输入。

分镜预处理器按 `entityManifest` 取得本集已绑定资产，而不是扫描全项目。它只创建外观显著变化的子资产，创建结果标为无图待补。已有 `expression`、`variant`、`initial` 子图必须保留在后续快照中。

分镜 Agent 对每个镜头可选择同一主资产下的多个 `AssetItem.id`，也可在默认主场景之外追加多个场景 `AssetItem.id`。数据层保持 `sceneAssetItemId` 为主场景，并以 `sceneAssetItemIds` 保存有序全集；资产引用同时返回兼容字段 `sceneRef` 与全部场景的 `sceneRefs`。生成时过滤无图子项，并遵守模型 `maxReferenceImages`。

## 数据与接口改动

无需数据库迁移。新增候选 DTO/服务和工具；在 `SceneEntity` JSON 中使用可选字段 `selectedAssetId` 仅作为工具输入，持久化后仍以 `assetId`、`assetItemId`、`source` 为准。

`GetStoryboardSceneItemsToolExecutor` 的引用结果新增：`assetId`、`assetName`、`itemType`、`hasImage`。这保持现有字段兼容。

## 失败处理与可观测性

- 候选为空：仅 core/supporting 自动补建，返回 `auto_created_episode_catalog` 和 `hasImage=false`。
- 候选歧义：不猜测，不创建；返回候选列表与 `ambiguous_episode_catalog`，让 Agent 选择后重试。
- AI 传入错误候选 ID：拒绝保存。
- 镜头引用无图子项：允许资产绑定，但引用查询明确 `hasImage=false`；生成 Agent 不将该 URL 传给模型，并报告缺少参考图。
- 每个 resolve 工具结果返回按 `source` 聚合的数量，便于界面显示“精确复用/后缀复用/待确认/补建”。

## 验收标准

1. 第 1 集存在 `凌炽表情图`、`凌炽三视图` 时，解析实体“凌炽”能以 `suffix_normalized` 唯一复用当前集资产，不创建第二个“凌炽”。
2. 第 2 集存在同名候选时，第 1 集解析绝不使用它。
3. 两个当前集候选同分时，结果为歧义且不创建资产；显式选择其中一个后才绑定。
4. 完整剧本解析不再创建 `episodeNumber = NULL` 的 AI 主资产。
5. 分镜快照创建于场次绑定与变体预处理之后，包含当前集最终子资产。
6. 单镜头可保留同一主资产的两张子图；查询镜头资产引用时两张图均带同一父资产信息。
7. 有图子项进入首尾帧/视频参考列表；无图占位子项不作为图片 URL 传入。
8. 后端相关单元测试、前端构建和受影响回归全部通过。
9. 一个镜头未显式追加场景时仍只有默认主场景；显式追加后主场景不变，所有有图场景都可被下游作为参考图读取。
