# 生成型分镜时长统一设计

## 背景

现有分镜编写器采用传统影视剪辑规则：每个场次通常拆为 3–15 个镜头、每镜 2–10 秒。剧情素材流程却会针对每个镜头生成一张“15 秒 25 宫格”，而视频工具在未显式收到时长时默认生成 5 秒。这导致剪辑镜头时长、25 宫格时长和实际视频时长不一致。

本设计把分镜条目定义为“可独立生成的视频镜头”，而不是最细粒度的后期剪辑切点。

## 目标

- 新生成的剧情分镜默认是 12–15 秒的连续生成镜头。
- 一个镜头内部必须有起始状态、事件推进和结束状态，可由一条连续运镜自然表达。
- 同一个 `StoryboardItem.duration` 同时作为分镜预算、25 宫格时间范围和 `generate_video` 请求时长。
- 25 宫格只服务时长不少于 12 秒的剧情镜头，且严格按该镜头的实际时长生成，不再硬编码为 15 秒。
- 时长永远不超过当前默认视频模型的能力范围。

## 非目标

- 不自动修改或合并历史分镜；已有短镜头应由用户重新生成或人工重写。
- 不强制所有镜头达到 12 秒。正反打、时空跳转、插入特写和无法连续表达的内容仍可为短镜头。
- 不改变战斗素材流程；战斗镜头继续使用动作故事板，而非剧情 25 宫格。

## 分镜规则

分镜编写器在开始写镜头前查询默认视频模型能力，并以 `maxDuration` 作为上限：

| 模型最大时长 | 剧情镜头默认时长 |
| --- | --- |
| 至少 15 秒 | 12–15 秒 |
| 12–14 秒 | 12 秒到模型上限 |
| 小于 12 秒 | 使用模型支持的短镜头时长；不默认生成 25 宫格 |
| 未返回最大时长 | 使用模型 `defaultDuration`；不默认生成 25 宫格 |

合并相邻内容时，必须同时满足：同一场景/时间、主体和空间关系连续、同一事件目标，并可通过连续调度或运镜完成。合并后需要重写为一个连续动作过程，不能把原来的多个镜头描述或硬切指令简单串接。

例如“黑屏出现裂缝 → 冷白光渗出 → 裂缝蔓延 → 帷幕与空间轮廓显现”应成为一条 12–15 秒的连续镜头；“角色 A 说话 → 切角色 B 反应 → 切回 A”若不能通过同框调度自然完成，则维持多个短镜头。

## 数据与调用约定

不新增表或字段。`StoryboardItem.duration` 统一代表该分镜条目的目标视频时长，并保存为整数秒。

```text
剧本场次
  -> 连续视频镜头（StoryboardItem, duration=12..15）
  -> 同时用于 25 宫格的叙事时间轴
  -> generate_video(duration=StoryboardItem.duration)
```

视频执行器必须读取镜头的 `duration`，检查其是否位于当前模型的 `minDuration` 与 `maxDuration` 内，再显式传给 `generate_video`。生成工具不再为 Agent 静默补充 5 秒，避免改变创作意图。

## 25 宫格规则

- 仅剧情模式且 `duration >= 12` 的连续镜头可默认生成 25 宫格。
- 25 格表示连续叙事节拍，不表示 25 次强制剪切；12 秒镜头的平均节拍约为 0.48 秒，15 秒镜头约为 0.6 秒。
- 默认提示词必须引用该镜头的真实秒数，例如“生成覆盖此 12 秒镜头的 25 宫格”。
- 短镜头仍可使用首帧、尾帧或普通故事板图，但不默认进入 25 宫格流程。

## 验收标准

1. 分镜 Agent 能获得默认视频模型的时长能力，并按能力生成连续镜头。
2. 新剧情镜头在支持的模型下默认时长为 12–15 秒，且内容包含开始、发展和收束。
3. 25 宫格默认提示词使用条目的实际时长；短于 12 秒时前端和 Agent 都提示其不适用。
4. 分镜视频执行器向 `generate_video` 显式传入条目时长。
5. 后端拒绝小于模型最小时长、超过模型最大时长或非正数的视频任务。
