# 分集缺失资产自动补建设计

## 目标

用户上传的分集资产是视觉参考的优先来源，但不要求覆盖剧本中的全部实体。AI 在解析当前剧本集时，发现必要资产缺失，应补建到当前集并自动关联场次、分镜；不生成图片。

## 规则

- 先按 `(projectId, episodeNumber, type, name)` 精确匹配用户上传或此前 AI 补建的资产。
- 仅对保留范围内的 `core` 和 `supporting` 实体补建；`atmospheric` 和 `filtered_limit` 不补建。
- 缺失时创建 `sourceType=2` 的当前集资产，并复用资产服务自动创建的 `initial` 子资产。该子资产不写入图片 URL。
- manifest 将来源标记为 `auto_created_episode_catalog`，并保存主资产与初始子资产 ID。
- 同集同名同类型只创建一次；不同集同名仍各自独立。
- 分镜保存沿用现有同集校验。补建资产会作为场次核心默认实体自动继承到镜头；辅助实体仅在镜头明确选择时出现。

## 数据流

1. 分集解析器提交实体清单并携带 `scriptEpisodeId`。
2. 解析服务从该剧本集派生 `episodeNumber`，查询本集资产。
3. 查询命中则返回 `matched`；未命中则创建无图占位资产并返回 `auto_created_episode_catalog`。
4. 场次保存 manifest 与已解析的资产 ID。
5. 分镜读取场次默认实体并继承补建资产；保存端仍拒绝跨集资产 ID。

## 错误与展示

- 不会为占位资产生成图片，也不会把它当作用户上传图片。
- 资产面板显示“AI 补建”来源和所属集数；没有图片时使用既有类型占位展示。
- 创建失败时返回错误，不写入带半套 ID 的 manifest。

## 测试

- 缺失的核心/辅助实体在当前集创建一次，并带空图片 initial item。
- 同名资产在另一集存在时，仍在当前集创建独立资产。
- 氛围实体、超限实体不创建。
- 新补建资产可写入场次，并能作为分镜默认资产通过同集校验。
