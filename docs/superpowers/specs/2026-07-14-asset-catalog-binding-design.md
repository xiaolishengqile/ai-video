# 资产目录与三类场次绑定设计

## 目标

让已上传资产在剧本和分镜解析时可确定性复用；每个场次可同时绑定角色、场景、道具；界面明确展示绑定结果和来源。

## 决策

- 不新建资产快照表。项目资产本身是唯一目录；每个 Agent 在开始时读取一次 `list_project_assets`，分镜预处理后分镜 Agent 再读取一次最新目录。
- `entityManifest` 是场次绑定的唯一事实来源。旧的三个主资产字段由它派生，保留给兼容查询和 UI。
- 资产解析由后端按项目、类型、精确名称确定性复用。名称不一致时不猜测，保留现有自动创建行为。
- 一个场次的角色（至多 3）、场景（至多 1）、道具（至多 3）独立计数，可同时存在。
- 镜头继承核心实体；仅镜头专属变体或辅助实体显式指定子资产。

## 范围

1. 让实际运行的剧本场次子 Agent 使用 manifest 流程，移除旧字段直填规则。
2. 将资产目录读取和三类实体判断写入剧本/分镜提示词。
3. 使前端在场次和镜头详情中显示三类资产、实体来源和镜头绑定来源。
4. 以自动化测试覆盖三类资产同时绑定及提示词运行时覆盖关系。

## 非目标

- 不做图片视觉识别、别名持久化、人工确认弹窗或历史数据回填。
- 不把项目全部资产绑定到每一个场次或镜头。
